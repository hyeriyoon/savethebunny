Game Plan

The attached game templates are a starting point, but how you use your acquired JavaScript skills in building your game is up to you.

Having a game plan is MANDATORY, it will account for 10% of your final grade.

In essence, your game plan acts as both a blueprint of what you will build as well as a a breakdown of the elements that you'll build it with.

A successful Game Plan should include, at minimum:

  1. A text description of game design and high level game functionality
  2. A screen-by-screen description of the UI
    1. Identify as many HTML elements as possible 
  3. A breakdown of the functional requirements
    • See provided game templates 
  4. A description chart of primary game loop(s) and/or logic flow
    • What happens each turn if turn based or each loop if time driven?
  5. Glossary of Variables
    1. name
    2. description
  6. Glossary of Functions
    1. name
    2. description
    3. parameters
  7. Glossary of Objects
    1. Properties
    2. Methods

Though you must submit your game plan before Day 06 (June 17), its primary purpose is to guide you from concept development through to completion of your project.  

It is intended to be a living document that evolves as you proceed through your build.  

Although you will be expected to meet the high-level functional requirements that you define in your game plan, everything else about it is fluid and your own.
